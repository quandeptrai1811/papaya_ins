# Papaya Insurance Engineering Challenges

Welcome to the Papaya Insurance Engineering repository! This repository contains a series of AI Engineering challenges, showcasing full-stack capabilities, data processing, and UI/UX engineering.

## Project Structure

The repository is divided into two main categories:

- **`AI_Engineering_Challenges/`**: Contains practical, hands-on engineering projects.
- **`Logical_questions/`**: Contains algorithmic and logical problem-solving questions.

## Completed Challenges

### [Challenge 01: Responsive Landing Page](./AI_Engineering_Challenges/AI_Challenge_01)
A responsive, modern UI/UX landing page redesign. 
- **Tech Stack**: HTML, CSS.
- **Highlights**: Features a responsive grid system, premium styling, and ensures consistent rendering across Desktop, Tablet, and Mobile devices.

### [Challenge 02: Claims Data Cleanup & Reporting](./AI_Engineering_Challenges/AI_Challenge_02)
A data engineering pipeline to clean and normalize messy insurance claim datasets.
- **Tech Stack**: Python (Standard Library).
- **Highlights**: Handles duplicates, corrects typos and casing, normalizes dates to ISO 8601, validates numerical data, and automatically generates a comprehensive markdown report (`report.md`) detailing the data quality metrics.

### [Challenge 03: Notification Email Templates & Next.js Previewer](./AI_Engineering_Challenges/AI_Challenge_03)
A full-stack suite of premium HTML email templates and a web application to preview them.
- **Tech Stack**: Next.js (App Router), React, CSS, HTML.
- **Highlights**: Features 6 meticulously designed email templates for the entire claims lifecycle. The Next.js web application allows operations teams to dynamically preview the templates with real-time variable interpolation across simulated mobile and desktop viewports.

## How to Run
Each challenge contains its own specific instructions and requirements. Please navigate to the individual challenge directories to view their respective `README.md` files for detailed setup and execution instructions.
